## Honest Feedback

**Friction 1: Night-to-day theme thrashing.** The 3D train went through multiple iterations — night theme, then day, then camera angles (on-top, cinematic, monitor, free orbit). Each change was a commit. The source in ψ/lab got out of sync with docs/. We caught it in the follow-up session but this is a workflow problem: source of truth should be clear from the start.

**Friction 2: Oracle birth counting is messy.** Issue search returned 30+ birth announcements with duplicates, encoding issues (mojibake in Thai), and multiple attempts per student. Counting "20+ unique Oracles" required manual deduplication. A structured registry with unique IDs would make this trivial. Issue #60 exists but wasn't updated fast enough to keep up with the wave.

**Friction 3: Can't preview web pages.** Built 3 HTML pages (map-real, dustboy, train variants) without seeing them. The DustBoy page needed 3 iterations just to get the layout right (raw JSON vs PM2.5 summary, log position). A browser preview tool would have saved at least 3 commits of "fix layout" churn.

## Lessons Learned

1. **Workshop materials must be self-contained** — Students need a single URL (GitHub Pages) with everything: guide, examples, data. The 4-step guide + live demos + public CSV worked well.
2. **Real data makes teaching stick** — Province boundaries + river geometry from OSM, live PM2.5 from MQTT. Students engage more when data is real, not mock.
3. **Students surprise you** — "0 retros" became 10+ without prompting. The act of creating an Oracle gave them ownership to write.
