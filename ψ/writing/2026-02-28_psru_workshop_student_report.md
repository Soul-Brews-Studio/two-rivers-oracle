## 1. ขอบเขตงานและสิ่งที่ได้ทำในวันนี้ (What We Did)

ในกิจกรรม Workshop ตลอดช่วงเช้าถึงบ่าย สิ่งที่ได้ลงมือปฏิบัติจริง มีดังนี้:
1. **ออกแบบและสร้าง CV / Portfolio ส่วนตัว**: 
   - เริ่มต้นจากการใช้ AI (Gemini Deep Research) เพื่อร่างข้อมูลส่วนตัว
   - ย้ายข้อมูลมารวบรวมใน Google Docs จากนั้นแปลงเนื้อหาให้อยู่ในรูปแบบ Markdown 
   - ใช้เครื่องมือ AntiGravity (AI Agent) เพื่อต่อยอดแปลงโครงสร้างเหล่านั้นเป็นหน้าเว็บไซต์ HTML ที่สมบูรณ์
2. **สร้างระบบสมองที่สอง (Oracle ของตัวเอง)**:
   - เข้าใจโครงสร้างของ AI ในฐานะผู้ช่วยส่วนตัว (Oracle)
   - ทำการเปิดตัว (Birth Announcement) บน GitHub Issues โดยเลือกกำหนดชื่อ, ธีม และเป้าหมายของ AI ของตัวเอง (เช่น การตั้งชื่อ Oracle และปรัชญาของตนเอง)
3. **อัปโหลดเว็บไซต์ขึ้น GitHub Pages**:
   - นำไฟล์ HTML ที่พัฒนาเสร็จแล้ว ไป Host ทิ้งไว้บน GitHub Pages (รูปแบบ `username.github.io`)
   - ทำให้สุดท้ายทุกคนมีเว็บไซต์ Portfolio ของตนเองที่เชื่อมต่อบนโลกออนไลน์ได้ทันที
4. **ทำแบบฝึกหัดการดึงข้อมูลผ่าน GitHub API**:
   - เขียนสคริปต์ (Bun + Shell Script) เพื่อสแกนดูเพื่อนๆ ในรุ่นที่เพิ่งประกาศสร้าง Oracle 
   - ฝึกแก้ปัญหา JSON Parsing เวลาดึง API มาแสดงผลเป็นตาราง

## 2. ปัญหา อุปสรรค และความยากลำบากที่พบเจอ (Challenges & Problems)

ตลอดการทำ Workshop เป็นช่วงเวลาที่ดุเดือดและท้าทายมาก ติดปัญหาและต้องแก้ไขข้อผิดพลาดหลายจุด:
1. **ปัญหา Characters Encoding ในภาษาไทย**: ตอนสร้าง Issue บน GitHub ผ่าน Command Line (CLI) เจอปัญหาภาษาไทย Error หรือประมวลผลเพี้ยน ทำให้บางคนสร้าง Issue ซ้ำเผลอเบิ้ลไป 3-4 รอบ ต้องมานั่งลบทิ้งและหาวิธีหลบหลีก
2. **ปัญหาเกี่ยวกับ Syntax และ Script วนลูป (JSON Parsing)**: ช่วงที่เขียนสคริปต์ดึง API เจอข้อจำกัดของ Shell Script (Heredoc `EOF`) พิมพ์เครื่องหมาย Quote หรือ String ไม่ครบทำให้ Script พังกลางคัน บางครั้ง API ส่งไฟล์กลับมาเป็น Text ไม่ใช่ JSON ทำให้อ่านค่าไม่ได้ ต้องมาดัก Try/Catch หาสาเหตุกันทีละบรรทัด
3. **ความสับสนจากการสลับ Context ไปมา (Context Switching)**: ต้องเปลี่ยนกลับไปมาระหว่างโปรแกรมหลายตัว (Terminal, แลกเปลี่ยนไฟล์ผ่าน Google Docs, เขียน Code Markdown บน Text Editor, เปิดเช็คผลบน Browser และเข้า GitHub) ทำให้สมองรับภาระหนักมากในการปะติดปะต่อว่ากำลังทำขั้นตอนไหนอยู่
4. **ความอ๊องจากการเชื่อมโยง Repo ต่างๆ**: การโยง Issue ข้าม Repository ของ GitHub อัตโนมัติ ทำให้ผู้ใช้หลงทางและหน้าจอ Terminal ลิงก์ไปยัง Issue ผิดที่ ทำให้เหนื่อยกับการต้องพิมพ์ Full URL ด้วยตัวเองตลอดเวลา

## 3. ความรู้สึกเหนื่อยล้าและอุปสรรคทางกายภาพ (Exhaustion & Mental Load)

ยอมรับว่าการเรียนรู้เรื่องทั้งหมดนี้รวดเดียวตั้งแต่ 8 โมงเช้าเกือบถึงบ่ายสอง เป็นสิ่งที่ **เหนื่อยล้าทางสมอง (Cognitive Overload) มากๆ** 
เราต้องคิดตั้งแต่เรื่องการ Design ข้อมูล Portfolio ตัวเองให้ออกมาดูดี ไปจนถึงการต้องคิดลอจิกในการเขียน Code ซึ่งใช้สมองคนละซีกกัน การจับเรื่องนามธรรมเช่น "ปรัชญาการทำงานของ AI (Form and Formless)" มาประกอบเป็นเรื่อง Software Development ทำให้รู้สึกเหมือนสมองทำงานแบบ Full-load เกือบ 5 ชั่วโมงเต็ม ไม่รวมความเหนื่อยล้าสะสมจากการเดินทาง และการต้องโฟกัสหน้าจอคอมพิวเตอร์ที่เต็มไปด้วย Error Logs ตลอดเวลา จนถึงท้ายที่สุด แม้จะสามารถ Deploy เว็บไซต์และสร้าง Oracle ตัวเองจนสำเร็จ แต่ก็กินพลังงานชีวิตจนแทบหมดหลอด ถือเป็นการลงมือทำที่คุ้มค่า แต่ก็ต้องแลกมากับความล้าทางจิตใจขั้นสุดเช่นกัน
